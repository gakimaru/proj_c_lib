【開発環境サンプル】

■proj_c_lib

--------------------------------------------------------------------------------
▼内容

・プロジェクトC開発用ライブラリ用リポジトリ。

・リポジトリを分けているのは、プロジェクトの担当プログラマーが、
  処理のレベルを意識してソースコードを切り分けて扱えるようにするため。

・プロジェクト専用ライブラリなので、再利用性は重視せず、ヘッダーファイルと
  ソースファイルの両方をそのまま使用する。

--------------------------------------------------------------------------------
▼ブランチ

・本流のみ
※必要に応じてパッチ版や海外版などのブランチを作成。

--------------------------------------------------------------------------------
▼依存リポジトリ

・common_lib_individual_for_studio_x ... 共通ライブラリの動作設定（ヘッダーファイル）。
・common_lib                         ... 共通ライブラリ（ヘッダーファイル）。

--------------------------------------------------------------------------------
▼サブモジュール適用フォルダ

（なし）
